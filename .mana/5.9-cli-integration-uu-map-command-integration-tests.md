## What to build

Wire the manifest crate into the `uu` CLI as the `uu map` command, and add integration tests.

### Files to create

1. **crates/uu/src/cmd/map.rs** â€” the `uu map` CLI command:
   ```rust
   //! `uu map` â€” generate a project manifest.

   use std::path::PathBuf;
   use anyhow::Result;
   use clap::Args;

   #[derive(Args)]
   pub struct MapArgs {
       /// Project directory (default: current directory)
       path: Option<PathBuf>,

       /// Output file (default: .manifest.yaml)
       #[arg(short, long)]
       output: Option<PathBuf>,

       /// Output format: yaml, json
       #[arg(short, long, default_value = "yaml")]
       format: String,

       /// Write to stdout instead of file
       #[arg(long)]
       stdout: bool,

       /// Show detected language and frameworks without generating manifest
       #[arg(long)]
       detect_only: bool,

       /// Show diff against existing manifest
       #[arg(long)]
       diff: bool,

       /// Only run specific adapters (comma-separated)
       #[arg(long)]
       adapters: Option<String>,

       /// Show what would be generated without writing
       #[arg(short = 'n', long)]
       dry_run: bool,
   }
   ```

   `pub fn execute(args: MapArgs) -> Result<()>`:
   1. Resolve project path (default: cwd)
   2. If `--detect-only`: detect project kind, list matching adapters, exit
   3. Call `uu_manifest::generate(path)` to build manifest
   4. If `--diff`: load existing manifest from output path, compute diff, print
   5. If `--dry-run`: print what would be written, exit
   6. Serialize to YAML or JSON based on `--format`
   7. If `--stdout`: print to stdout
   8. Else: write to output file (default: `.manifest.yaml`)
   9. Print summary: "Generated .manifest.yaml (X types, Y functions, Z routes)"
