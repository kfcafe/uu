## Task
Add a new project-aware subcommand: `uu build`.

This should compile/build the project (without doing a global install), following the same conventions as `uu install`, `uu run`, and `uu test`.

## Implementation notes
- Implement as a project-aware command like the existing ones.
  - Supports -n/--dry-run.
  - Supports passthrough args after `--` (append to the LAST step, like install/run/test).
- New module: crates/uu/src/cmd/build.rs
  - fn steps(kind: &ProjectKind) -> Result<Vec<Step>> (or Vec<Step> if infallible)
  - pub(crate) fn execute(dry_run: bool, extra_args: Vec<String>) -> Result<()>

## Suggested command mapping
- Cargo: cargo build
- Go: go build ./...
- Elixir: mix compile
- Python: python -m build (OK to bail with a helpful message if the build module is missing)
- Node: <pm> run build (pm is bun/pnpm/yarn/npm)
- Gradle: ./gradlew build if wrapper else gradle build
- Maven: mvn package
- Ruby: either bail with a clear message, or try: bundle exec rake build
- Meson: meson setup builddir only if builddir/ does not exist, then meson compile -C builddir
- CMake: cmake -B build then cmake --build build
- Make: make

If you bail for an ecosystem, use an actionable error message (similar to how `uu run` treats CMake/Meson).
