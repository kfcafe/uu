## Task
Improve `crates/detect/src/lib.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] consistency — `crates/detect/src/lib.rs:75-83`
  - `detected_file()` always returns `"build.gradle"` for the `Gradle` variant, but detection on line 149-152 also matches `build.gradle.kts`. If only `build.gradle.kts` exists, the user sees "detected Gradle (build.gradle)" which is inaccurate and confusing.
  - Suggested fix: Store the actual detected file in the `Gradle` variant (e.g. `Gradle { wrapper: bool, kts: bool }`) and return the correct filename from `detected_file()`.
- [low] consistency — `crates/detect/src/lib.rs:198-211`
  - `supported_table()` lists 12 project files but omits `setup.cfg`, which `detect()` (line 132-137) recognizes as a valid Python project marker. A user in a `setup.cfg`-based Python project who gets the "no project detected" error (from a different directory) would not see `setup.cfg` in the supported files list.
  - Suggested fix: Add `("setup.cfg", "pip install .")` to the `entries` array in `supported_table()`.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
