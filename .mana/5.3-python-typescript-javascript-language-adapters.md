## What to build

Implement Python, TypeScript, and JavaScript language adapters for the manifest crate. Follow the same pattern established by the Rust adapter (see `crates/manifest/src/adapters/lang/rust.rs`).

### Files to create

1. **crates/manifest/src/adapters/lang/python.rs** â€” Python language adapter:
   - `pub struct PythonAdapter;`
   - `detect`: returns true when `ctx.kind == ProjectKind::Python { .. }`
   - `extract`: uses tree-sitter-python to parse `.py` files:
     - `class X:` â†’ TypeDef (kind: Class)
     - `class X(Base):` â†’ TypeDef with implements: ["Base"]
     - `def func():` at module level â†’ Function
     - `def _func():` â†’ Function with Private visibility
     - `async def` â†’ Function with is_async: true
     - `@property`, `@staticmethod`, `@classmethod` â†’ noted in methods
     - Module structure from directory/file layout
   - Filter: `.py` files, skip `__pycache__`, skip `test_*.py` / `*_test.py` for is_test detection
   - Priority: 100, Layer: Language

2. **crates/manifest/src/adapters/lang/typescript.rs** â€” TypeScript adapter:
   - `pub struct TypeScriptAdapter;`
   - `detect`: returns true when `ctx.kind == ProjectKind::Node { .. }` AND tsconfig.json exists in root
   - `extract`: uses tree-sitter-typescript to parse `.ts` and `.tsx` files:
     - `interface X` â†’ TypeDef (kind: Interface)
     - `type X = ` â†’ TypeDef (kind: TypeAlias)
     - `enum X` â†’ TypeDef (kind: Enum)
     - `class X` â†’ TypeDef (kind: Class)
     - `export function` â†’ Function with Public visibility
     - `export default function` â†’ Function
     - `export const x = () =>` â†’ Function (arrow function exports)
     - `async function` â†’ Function with is_async: true
   - Filter: `.ts`, `.tsx` files, skip `node_modules/`, `dist/`, `.d.ts` files
   - Priority: 100, Layer: Language

3. **crates/manifest/src/adapters/lang/javascript.rs** â€” JavaScript adapter:
   - `pub struct JavaScriptAdapter;`
   - `detect`: returns true when `ctx.kind == ProjectKind::Node { .. }` AND NO tsconfig.json
   - `extract`: uses tree-sitter-javascript to parse `.js`, `.jsx`, `.mjs`, `.cjs` files:
     - `class X` â†’ TypeDef (kind: Class)
     - `module.exports` / `export function` â†’ Function with Public visibility
     - Arrow function exports â†’ Function
   - Same filtering as TypeScript (skip node_modules, dist)
   - Priority: 99 (slightly lower than TS so TS wins when both could match)
   - Layer: Language
