## What to build

Implement the first two language adapters (Rust and Go) for the manifest crate. These establish the pattern that all other language adapters follow.

### Files to create

1. **crates/manifest/src/adapters/lang/rust.rs** — Rust language adapter:
   - `pub struct RustAdapter;`
   - `detect`: returns true when `ctx.kind == ProjectKind::Cargo`
   - `extract`: uses tree-sitter-rust to parse `.rs` files and extract:
     - `pub struct` → TypeDef (kind: Struct, with fields)
     - `pub enum` → TypeDef (kind: Enum, with variants as fields)
     - `pub trait` → TypeDef (kind: Trait, with method signatures)
     - `impl X for Y` → adds to implements list on the type
     - `pub fn` at module level → Function
     - `pub(crate) fn` → Function with Internal visibility
     - `mod` declarations → Module (with file mapping)
     - `#[test] fn` → Function with is_test: true
     - `async fn` → Function with is_async: true
   - Source references: `"src/auth.rs:12"` format
   - Filter: only process `.rs` files from `ctx.files`
   - Priority: 100 (language adapters are high priority)
   - Layer: Language

2. **crates/manifest/src/adapters/lang/go.rs** — Go language adapter:
   - `pub struct GoAdapter;`
   - `detect`: returns true when `ctx.kind == ProjectKind::Go`
   - `extract`: uses tree-sitter-go to parse `.go` files and extract:
     - `type X struct` → TypeDef (kind: Struct, with fields)
     - `type X interface` → TypeDef (kind: Interface, with methods)
     - Exported functions (capitalized) → Function with Public visibility
     - Non-exported functions → Function with Private visibility
     - Package declarations → Module
   - Source references: `"cmd/server/main.go:15"` format
   - Filter: only process `.go` files, skip `_test.go` files for non-test functions
   - Priority: 100
   - Layer: Language

### Files to modify

3. **crates/manifest/src/adapters/lang/mod.rs** — register both adapters:
   ```rust
   pub mod rust;
   pub mod go;
   ```

4. **crates/manifest/src/adapters/mod.rs** — add to `all_adapters()`:
   ```rust
   pub fn all_adapters() -> Vec<Box<dyn Adapter>> {
       vec![
           Box::new(lang::rust::RustAdapter),

   }
   ```

5. **crates/manifest/Cargo.toml** — add tree-sitter grammar deps:
   ```toml
   tree-sitter-rust = "0.23"
   tree-sitter-go = "0.23"
   ```
   Note: Check crates.io for the latest versions compatible with tree-sitter 0.24. The grammar crate versions may differ from tree-sitter core.

### Tree-sitter usage pattern

```rust
use tree_sitter::Parser;

fn parse_file(source: &[u8], language: tree_sitter::Language) -> Option<tree_sitter::Tree> {
    let mut parser = Parser::new();
    parser.set_language(&language).ok()?;
    parser.parse(source, None)
}
```

Then walk the tree with a `TreeCursor` or iterate `node.children()`, matching on `node.kind()` values like `"struct_item"`, `"function_item"`, `"enum_item"`, etc.

For Rust, key node kinds:
- `struct_item`, `enum_item`, `trait_item`, `impl_item`, `function_item`, `mod_item`
- `visibility_modifier` child on items indicates pub/pub(crate)

For Go, key node kinds:
- `type_declaration` → contains `type_spec` with `struct_type` or `interface_type`
- `function_declaration`, `method_declaration`
- `package_clause`

### Tests

For each adapter:
- Parse a sample source string and verify types/functions are extracted correctly
- Verify visibility detection (pub vs private)
- Verify source locations are correct (file:line format)
- Verify detect() returns true/false for correct ProjectKind
- Verify async functions are flagged (Rust)
- Verify test functions are flagged (Rust)
- Verify Go exported vs unexported distinction

### Important
- Make file paths in source references relative to project root
- Skip test files for Go when extracting (but still flag #[test] functions in Rust)
- The adapters should be resilient — if a file fails to parse, log and skip it, don't abort
