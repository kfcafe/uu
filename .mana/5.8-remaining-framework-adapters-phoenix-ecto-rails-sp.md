## What to build

Implement the remaining framework adapters for Elixir, Ruby, Java, Go, and .NET ecosystems.

### Files to create

1. **crates/manifest/src/adapters/framework/phoenix.rs** — Phoenix adapter:
   - `pub struct PhoenixAdapter;`
   - `detect`: `:phoenix` in mix.exs deps
   - `extract`: parse `router.ex` for route definitions:
     - `get "/path", Controller, :action` → Route + Endpoint
     - `resources "/path", Controller` → expanded CRUD routes
     - `pipe_through [:browser]` → pipeline info (middleware)
     - `scope "/api" do ... end` → prefix for nested routes
   - Simple line/regex parsing — Phoenix router is quite regular
   - Priority: 50, Layer: Framework

2. **crates/manifest/src/adapters/framework/ecto.rs** — Ecto adapter:
   - `pub struct EctoAdapter;`
   - `detect`: `:ecto` or `:ecto_sql` in mix.exs deps
   - `extract`: find Ecto.Schema modules:
     - `schema "table_name" do` block → DataModel (orm: "ecto")
     - `field :name, :type` → Field
     - `has_many`, `has_one`, `belongs_to`, `many_to_many` → Relation
   - Use tree-sitter-elixir or line parsing
   - Priority: 50, Layer: Framework

3. **crates/manifest/src/adapters/framework/rails.rs** — Rails adapter:
   - `pub struct RailsAdapter;`
   - `detect`: "rails" in Gemfile
   - `extract`:
     - `config/routes.rb`: `get '/path'`, `resources :name`, `namespace` blocks → Routes
     - `app/models/*.rb`: classes inheriting `ApplicationRecord` or `ActiveRecord::Base` → DataModel (orm: "activerecord")
     - `app/controllers/*.rb`: controller classes → noted
   - Rails conventions make this mostly filesystem-based
   - Priority: 50, Layer: Framework

4. **crates/manifest/src/adapters/framework/spring.rs** — Spring Boot adapter:
   - `pub struct SpringAdapter;`
   - `detect`: "org.springframework.boot" in build.gradle or pom.xml
   - `extract`:
     - `@RestController` / `@Controller` classes → controllers
     - `@GetMapping("/path")`, `@PostMapping`, `@RequestMapping` → Route + Endpoint
     - `@Entity` classes → DataModel (orm: "jpa")
   - Use tree-sitter-java to find annotations
   - Priority: 50, Layer: Framework

5. **crates/manifest/src/adapters/framework/gin.rs** — Gin adapter:
   - `pub struct GinAdapter;`
   - `detect`: "github.com/gin-gonic/gin" in go.mod
   - `extract`: find gin route registrations:
     - `r.GET("/path", handler)` → Route + Endpoint
     - `r.POST(...)`, `group.GET(...)` etc.
   - Use tree-sitter-go to find method calls on gin engine/group
   - Priority: 50, Layer: Framework

6. **crates/manifest/src/adapters/framework/gorm.rs** — GORM adapter:
   - `pub struct GormAdapter;`
   - `detect`: "gorm.io/gorm" in go.mod
   - `extract`: find struct types embedding `gorm.Model`:
     - `type User struct { gorm.Model; Name string; ... }` → DataModel (orm: "gorm")
   - Use tree-sitter-go
   - Priority: 50, Layer: Framework

7. **crates/manifest/src/adapters/framework/aspnet.rs** — ASP.NET adapter:
   - `pub struct AspNetAdapter;`
   - `detect`: "Microsoft.AspNetCore" in .csproj
   - `extract`:
     - `[ApiController]` classes → controllers
     - `[HttpGet("/path")]`, `[HttpPost]` attributes → Route + Endpoint
     - Entity Framework `DbSet<T>` properties → DataModel (orm: "efcore")
   - Use tree-sitter-c-sharp
   - Priority: 50, Layer: Framework

### Files to modify

8. **crates/manifest/src/adapters/framework/mod.rs** — add all modules
9. **crates/manifest/src/adapters/mod.rs** — register all in `all_adapters()`

### Detection helpers

For Elixir deps: read mix.exs, look for `{:phoenix, ...}` or `{:ecto, ...}` in deps list.
For Ruby deps: read Gemfile, look for `gem 'rails'` or `gem "rails"`.
For JVM deps: read build.gradle for `implementation 'org.springframework.boot:...'` or pom.xml.
For Go deps: read go.mod (`ctx.go_mod`) for `require` lines.
For .NET deps: read .csproj XML for PackageReference elements.

### Tests

For each adapter:
- Create minimal sample files (router.ex, routes.rb, etc.) and verify extraction
- Verify detection from dependency files
- Phoenix: scope nesting, resources expansion
- Ecto: schema fields and associations
- Rails: resources routes, model detection
- Spring: annotation detection, JPA entities
- Gin: route groups
- GORM: gorm.Model embedding
- ASP.NET: attribute-based routing

### Important
- Many of these use simple line/regex parsing rather than tree-sitter — that's fine when the file format is regular
- Phoenix router, Rails routes.rb, and Spring annotations all have well-defined patterns
- Be pragmatic about what to extract — handle the 80% case, don't try to handle every edge case
- For go.mod parsing, `ctx.go_mod` contains the raw string — parse `require` blocks
