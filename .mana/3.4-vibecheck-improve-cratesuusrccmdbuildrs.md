## Task
Improve `crates/uu/src/cmd/build.rs` based on vibecheck findings.

## Findings (top 20)
- [high] edge-cases — `crates/uu/src/cmd/build.rs:28-31`
  - Meson `steps()` always emits `meson setup builddir` as the first step. If `builddir/` already exists (e.g. from a previous build), `meson setup` fails with an error. The README says "only if builddir/ does not exist" but the code doesn't check. The same issue exists in `install.rs` lines 25-29.
  - Suggested fix: Either check for `builddir/` existence before emitting the setup step (requires passing the project dir into `steps()`), or use `meson setup --reconfigure builddir` when the directory exists, or add `--wipe` as a fallback.
- [low] graceful-degradation — `crates/uu/src/cmd/build.rs:15-16`
  - Python `build` step runs `python -m build` (or `uv run python -m build`) which requires the `build` PyPI package to be installed. If it's missing, Python gives an opaque "No module named build" error. The bean description said to "bail with a helpful message if the build module is missing" but no such check exists.
  - Suggested fix: Either pre-check with `command_on_path` / `python -c "import build"` and bail with an actionable message like `"python -m build requires the 'build' package — pip install build"`, or document this as a known limitation.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
