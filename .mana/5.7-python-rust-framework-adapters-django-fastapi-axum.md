## What to build

Implement framework adapters for Python and Rust ecosystems.

### Files to create

1. **crates/manifest/src/adapters/framework/django.rs** — Django adapter:
   - `pub struct DjangoAdapter;`
   - `detect`: "django" or "Django" in pyproject.toml `[project.dependencies]` or requirements.txt
   - `extract`:
     - **urls.py** files: parse `urlpatterns` for route definitions
       - `path('products/', views.product_list)` → Route (path: "/products/", handler: "product_list")
       - `path('api/', include('api.urls'))` → noted as prefix mount
     - **models.py** files: find classes inheriting `models.Model`:
       - `class Product(models.Model):` → DataModel (orm: "django")
       - `name = models.CharField(...)` → Field (type: "CharField")
       - `ForeignKey`, `ManyToManyField` → Relation
     - **views.py**: find view functions/classes (basic detection)
   - Can use tree-sitter-python for models/views, simple line parsing for urls.py
   - Priority: 50, Layer: Framework

2. **crates/manifest/src/adapters/framework/fastapi.rs** — FastAPI adapter:
   - `pub struct FastApiAdapter;`
   - `detect`: "fastapi" in pyproject.toml dependencies or requirements.txt
   - `extract`:
     - Decorated routes: `@app.get("/path")`, `@app.post("/path")`, `@router.get(...)` etc.
       → Route + Endpoint with method and path
     - Pydantic models: `class X(BaseModel):` → DataModel (orm: "pydantic") or TypeDef
     - Use tree-sitter-python to find decorated function definitions
   - Priority: 50, Layer: Framework

3. **crates/manifest/src/adapters/framework/axum.rs** — Axum adapter:
   - `pub struct AxumAdapter;`
   - `detect`: "axum" in Cargo.toml `[dependencies]`
   - `extract`:
     - Router definitions: `Router::new().route("/path", get(handler))` → Route + Endpoint
     - `.route("/path", post(handler).get(handler2))` → multiple endpoints
     - Handler function signatures (already extracted by Rust language adapter as Functions)
     - Uses `ctx.cargo_toml` for detection
   - Use tree-sitter-rust to find Router method chains
   - Priority: 50, Layer: Framework

### Files to modify

4. **crates/manifest/src/adapters/framework/mod.rs** — add modules
5. **crates/manifest/src/adapters/mod.rs** — register in `all_adapters()`

### Detection helpers

For Python dependency detection, you'll need to check:
- `pyproject.toml`: parse `[project.dependencies]` array or `[tool.poetry.dependencies]` table
- `requirements.txt`: line-by-line, each line is a package name (possibly with version specifier)
- `ctx.package_json` won't help here — you may need to add similar caching for pyproject.toml to ProjectContext, or just read the files directly in the adapter

For Rust dependency detection:
- Check `ctx.cargo_toml` (a parsed TOML value) for `[dependencies].axum`

### Tests

- Django: create urls.py and models.py samples, verify routes and models extracted
- FastAPI: create sample with @app.get decorators and Pydantic models, verify routes/endpoints/models
- Axum: create sample with Router::new().route() chains, verify routes and endpoints
- Detection tests for each (matching and non-matching dependency files)

### Important
- Django urls.py can be complex (regex patterns, includes) — handle common cases, skip exotic ones
- FastAPI decorators can have dependencies, response_model etc. — extract path and method, skip the rest
- Axum Router chains can be deeply nested — handle the common `.route(path, method(handler))` pattern
