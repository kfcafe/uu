## Task
Add a new project-aware subcommand: `uu fmt`.

This command should run the ecosystem's formatter in "write" mode (i.e. it may modify files), similar to how `uu test` runs the ecosystem test runner.

## CLI shape
- Use the existing ProjectArgs style:
  - -n/--dry-run
  - passthrough args after `--` (append to the LAST step)

## Implementation notes
- New module: crates/uu/src/cmd/fmt.rs
- Main wiring:
  - crates/uu/src/main.rs: add Commands::Fmt(ProjectArgs)
  - crates/uu/src/cmd/mod.rs: export fmt module
- Consider adding a Clap alias `format` for convenience.

## Suggested command mapping
Keep it best-effort and bail with a helpful message where there is no obvious default:
- Cargo: cargo fmt
- Go: gofmt -w .
- Elixir: mix format
- Python: if `ruff` is on PATH, run `ruff format .`; else if `black` is on PATH, run `black .`; else bail with an actionable message
- Node: <pm> run format (pm is bun/pnpm/yarn/npm)

Unsupported ecosystems should return an error that explains what to run instead.
