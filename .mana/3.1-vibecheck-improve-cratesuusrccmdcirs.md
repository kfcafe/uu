## Task
Improve `crates/uu/src/cmd/ci.rs` based on vibecheck findings.

## Findings (top 20)
- [critical] edge-cases — `crates/uu/src/cmd/ci.rs:25-28`
  - `gofmt -l .` always exits 0 regardless of whether files need formatting. It merely lists unformatted files to stdout. The CI pipeline step will silently pass even when Go code is unformatted, defeating the purpose of a format check in CI.
  - Suggested fix: Replace `step("gofmt", &["-l", "."])` with a shell invocation that fails on non-empty output, e.g. `sh -c 'test -z "$(gofmt -l .)"'`, or use `gofmt -d .` and pipe through a check. Alternatively, use `goimports -l .` with similar non-empty check.
- [medium] edge-cases — `crates/uu/src/cmd/ci.rs:37-52`
  - Python `uv: true` CI branch checks `command_on_path("ruff")` to decide the pipeline, but when using `uv`, tools like `ruff` are typically installed in the project's virtualenv and may not be on the system PATH. The commands run `uv run ruff ...` which would find ruff in the venv, but the `command_on_path` gate prevents reaching those commands. Same pattern in `fmt.rs` lines 14-25 and `lint.rs` lines 14-24.
  - Suggested fix: For the `uv: true` branch, either always emit the `uv run ruff ...` steps (let `uv run` fail gracefully if ruff isn't installed), or probe with `uv run ruff --version` instead of `command_on_path("ruff")`.
- [medium] edge-cases — `crates/uu/src/cmd/ci.rs:20-20`
  - The Go CI pipeline uses `gofmt -l .` which lists unformatted files but always exits 0 regardless of whether files need formatting. This means the Go formatting check never actually fails the CI pipeline — it prints file paths to stdout but `uu ci` treats it as a pass. The Rust (`cargo fmt --check`) and Elixir (`mix format --check-formatted`) equivalents correctly exit non-zero when formatting issues exist.
  - Suggested fix: Replace `step("gofmt", &["-l", "."])` with a shell-based check that fails on output, such as `step("sh", &["-c", "test -z \"$(gofmt -l .)\""])`, or use `goimports -l .` which has the same issue but is more commonly wrapped. Alternatively, check for `golangci-lint` on PATH and use `golangci-lint run --disable-all --enable gofmt` which does exit non-zero.
