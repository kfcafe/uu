## What to build

Implement framework adapters for the Node.js ecosystem. These detect frameworks from package.json dependencies and extract semantic information (routes, models, components, auth).

### Files to create

1. **crates/manifest/src/adapters/framework/nextjs.rs** — Next.js adapter:
   - `pub struct NextJsAdapter;`
   - `detect`: "next" in package.json dependencies or devDependencies
   - `extract`:
     - App Router: scan `app/` directory for file conventions:
       - `page.tsx` / `page.jsx` / `page.ts` / `page.js` → Route (type: Page)
       - `layout.tsx` → Route (type: Layout)
       - `route.ts` → Route (type: ApiRoute), extract exported HTTP methods (GET, POST, etc.)
       - `loading.tsx`, `error.tsx`, `not-found.tsx` → noted
       - Directory structure maps to URL paths: `app/products/[id]/page.tsx` → "/products/[id]"
     - Also check `pages/` directory for Pages Router (legacy):
       - `pages/about.tsx` → Route (path: "/about", type: Page)
       - `pages/api/users.ts` → Route (path: "/api/users", type: ApiRoute)
     - `middleware.ts` at root → Route (type: Middleware)
   - Priority: 50, Layer: Framework

2. **crates/manifest/src/adapters/framework/express.rs** — Express adapter:
   - `pub struct ExpressAdapter;`
   - `detect`: "express" in package.json dependencies
   - `extract`: use tree-sitter to find method calls:
     - `app.get('/path', handler)` → Route + Endpoint
     - `app.post(...)`, `app.put(...)`, `app.delete(...)` → same
     - `router.get(...)` etc. → same (express Router)
     - `app.use('/prefix', router)` → middleware mounting (combine prefix)
   - Priority: 50, Layer: Framework

3. **crates/manifest/src/adapters/framework/prisma.rs** — Prisma adapter:
   - `pub struct PrismaAdapter;`
   - `detect`: `prisma/schema.prisma` file exists (or `schema.prisma` at root)
   - `extract`: simple line-by-line parser (Prisma schema is regular):
     - `model X {` → DataModel (orm: "prisma")
     - Fields: `name Type @...` → Field
     - Relations: field with `@relation(...)` → Relation
     - `@@index([...])` → indexes
     - `enum X {` → TypeDef (kind: Enum)
   - No tree-sitter needed — line parsing is sufficient
   - Priority: 50, Layer: Framework

4. **crates/manifest/src/adapters/framework/shadcn.rs** — shadcn/ui adapter:
   - `pub struct ShadcnAdapter;`
   - `detect`: `components.json` exists (shadcn config file)
   - `extract`: scan `components/ui/` directory for `.tsx` files:
     - Each file → Component (name from filename, e.g., "button", "dialog")
     - Optionally parse exported component props for `props` field
   - Priority: 30, Layer: Framework

5. **crates/manifest/src/adapters/framework/authjs.rs** — Auth.js/NextAuth adapter:
   - `pub struct AuthJsAdapter;`
   - `detect`: "@auth/core" or "next-auth" in package.json dependencies
   - `extract`: look for auth config file (`auth.ts`, `auth.js`, `[...nextauth]` route):
     - Provider names from config (e.g., "google", "github", "credentials")
     - Session strategy if detectable
   - Populates: `auth` field on manifest
   - Priority: 30, Layer: Framework
