## Task
Improve `crates/detect/src/lib.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] graceful-degradation — `crates/detect/src/lib.rs:162-170`
  - `command_on_path` uses the `which` command, which is not a POSIX standard and doesn't exist on Windows. This means `uv` detection for Python and all tool-availability checks (ruff, black, flake8) will always return `false` on Windows.
  - Suggested fix: Use `std::process::Command::new(name).arg("--version")` with null output, or use the `which` crate for cross-platform lookup. Alternatively, simply try to find the executable on `PATH` by iterating `std::env::split_paths(&std::env::var("PATH"))`.
- [low] edge-cases — `crates/detect/src/lib.rs:95-100`
  - Python detection matches `pyproject.toml`, `setup.py`, or `setup.cfg`, but `detected_file()` always returns `"pyproject.toml"` regardless of which file actually triggered the match. The user sees "detected Python (pyproject.toml)" even when only `setup.py` exists.
  - Suggested fix: Store the actual detected file in the `Python` variant: `Python { uv: bool, file: &'static str }` and return it from `detected_file()`.
- [low] edge-cases — `crates/detect/src/lib.rs:144-150`
  - `command_on_path()` spawns a subprocess (`which`) to check if a command exists. This is called at detection time for Python (`uv`) and at step-generation time for Python CI/fmt/lint (`ruff`, `black`, `flake8`). Each call forks a process, which is slow and could fail on minimal systems where `which` isn't installed (e.g., Alpine Linux containers without `which` package). The function silently returns `false` on failure, which means it would incorrectly report that `uv` is not available if `which` itself is missing.
  - Suggested fix: Replace the `which` subprocess with a pure-Rust PATH lookup (iterate `env::var("PATH")` segments and check if `Path::join(segment, name).is_file()` with execute permission). This is faster, more portable, and doesn't have the `which`-not-found edge case. The `which` crate on crates.io does exactly this.
