## Task
Add robust Python command fallback so `uu` works on machines that have `python3`/`pip3` but not `python`/`pip`.

This came up on macOS where `python` and `pip` are often absent by default, but `python3` and `pip3` exist.

## Desired behavior
For Python projects when `uv` is NOT available:
- `uu run` should use `python3` if `python` is missing.
- `uu install` should use `pip3` if `pip` is missing.
- (Optional but nice) `uu test` should prefer `pytest` if present; otherwise run `python3 -m pytest` if possible.

## Implementation notes
- Keep changes minimal.
- Do NOT add new dependencies.
- Prefer a small helper in `crates/uu/src/runner.rs` (or a new small module) that can choose the first available program from a list by scanning `$PATH` directly (do not shell out to `which`).
  - This makes tests deterministic because they can set PATH to a temp dir.

Example helper idea:
- `fn first_on_path(candidates: &[&str]) -> Option<String>`
- `fn pick_program(preferred: &[&str], fallback: &str) -> String`

Then update:
- `crates/uu/src/cmd/run.rs` (Python uv:false)
- `crates/uu/src/cmd/install.rs` (Python uv:false)
- `crates/uu/src/cmd/test_cmd.rs` (Python uv:false) if you implement the optional pytest fallback.

## Tests
Add an integration test in `crates/uu/tests/cli.rs` that proves the fallback works regardless of the developer machine:
1) Create a temp Python project with `pyproject.toml` + `main.py`.
2) Create a temp `bin/` directory with stub executables:
   - `python3` (exits 0)
   - `pip3` (exits 0)
   - and intentionally NO `python` / `pip`.
3) Run `uu run` and `uu install` with `PATH` set to ONLY that bin directory.
4) Assert both commands exit 0.

(If you also implement pytest fallback, add a stub `pytest` or assert `python3 -m pytest` is attempted.)

## Docs
Update README.md (small note under Python) that `uu` will fall back to `python3`/`pip3` when needed.
