## What to build

Create the `crates/manifest/` crate with all core infrastructure that adapters depend on.

### Files to create

1. **crates/manifest/Cargo.toml** — new crate `uu-manifest` with deps:
   - `uu-detect = { path = "../detect" }`
   - `serde = { version = "1", features = ["derive"] }`
   - `serde_json = "1"`
   - `serde_yaml = "0.9"`
   - `anyhow = "1"`
   - `ignore = "0.4"` (for gitignore-aware file walking)
   - `tree-sitter = "0.24"` (needed later by adapters, add now)
   - `[dev-dependencies]`: `tempfile = "3"`, `pretty_assertions = "1"`

2. **crates/manifest/src/lib.rs** — public API:
   - `pub mod schema;`
   - `pub mod context;`
   - `pub mod diff;`
   - `pub mod adapters;`
   - Re-export key types: `Manifest`, `ManifestFragment`, `ProjectContext`, `Adapter`
   - `pub fn generate(root: &Path) -> Result<Manifest>` — main entry point:
     1. Detect project kind via `uu_detect::detect`
     2. Build `ProjectContext`
     3. Run adapter registry to get matching adapters
     4. Call `extract` on each, collect `ManifestFragment`s
     5. Merge fragments into `Manifest`
     6. Return manifest

3. **crates/manifest/src/schema.rs** — all serde-serializable types:
   ```rust
   pub struct Manifest {
       pub project: ProjectMeta,
       pub types: BTreeMap<String, TypeDef>,
       pub functions: BTreeMap<String, Function>,
       pub modules: BTreeMap<String, Module>,
       pub routes: BTreeMap<String, Route>,
       pub endpoints: BTreeMap<String, Endpoint>,
       pub models: BTreeMap<String, DataModel>,
       pub auth: Option<AuthConfig>,
       pub components: Vec<Component>,
       pub integrations: Vec<Integration>,
   }

   pub struct ProjectMeta {
       pub name: String,
       pub kind: String,
       pub frameworks: Vec<String>,
       pub generated_at: String,
       pub uu_version: String,
   }

   pub struct ManifestFragment {
       pub types: BTreeMap<String, TypeDef>,
       pub functions: BTreeMap<String, Function>,
       pub modules: BTreeMap<String, Module>,
       pub routes: BTreeMap<String, Route>,
       pub endpoints: BTreeMap<String, Endpoint>,
       pub models: BTreeMap<String, DataModel>,
       pub auth: Option<AuthConfig>,
       pub components: Vec<Component>,
       pub integrations: Vec<Integration>,
   }

   pub struct TypeDef { name, source, kind: TypeKind, fields: Vec<Field>, methods: Vec<String>, visibility: Visibility, implements: Vec<String> }
   pub enum TypeKind { Struct, Class, Interface, Enum, Trait, Protocol, Union, TypeAlias }
   pub struct Function { name, source, signature, visibility: Visibility, is_async: bool, is_test: bool }
   pub struct Module { path, file, exports: Vec<String>, imports: Vec<String> }
   pub struct DataModel { name, source, orm: String, fields: Vec<Field>, relations: Vec<Relation>, indexes: Vec<String> }
   pub struct Route { path, file, route_type: RouteType, methods: Vec<String>, handler: Option<String> }
   pub enum RouteType { Page, Layout, ApiRoute, Controller, Middleware }
   pub struct Endpoint { path, file, method: String, handler: String, middleware: Vec<String> }
   pub struct Field { name: String, type_name: String, optional: bool }
   pub struct Relation { name: String, kind: RelationKind, target: String }
   pub enum RelationKind { HasOne, HasMany, BelongsTo, ManyToMany }
   pub struct Component { name: String, file: String, props: Vec<Field> }
   pub struct Integration { name: String, kind: String, config_file: Option<String> }
   pub struct AuthConfig { providers: Vec<String>, strategy: String }
   pub enum Visibility { Public, Private, Internal }
   ```
   All types derive `Debug, Clone, Serialize, Deserialize, Default` where appropriate.
   Use `#[serde(skip_serializing_if = "...")]` for optional/empty fields to keep YAML clean.

4. **crates/manifest/src/context.rs** — `ProjectContext` builder:
   ```rust
   pub struct ProjectContext {
       pub root: PathBuf,
       pub kind: ProjectKind,
       pub files: Vec<PathBuf>,
       pub package_json: Option<serde_json::Value>,
       pub cargo_toml: Option<toml::Value>,  // or serde_json::Value from parsing
       pub go_mod: Option<String>,
   }
   ```
   - `ProjectContext::build(root: &Path, kind: &ProjectKind) -> Result<Self>`
   - Walk source files respecting `.gitignore` (use `ignore` crate's `WalkBuilder`)
   - Cache parsed dependency files (package.json, Cargo.toml, go.mod, etc.)

5. **crates/manifest/src/diff.rs** — manifest diffing:
   - `pub fn diff(old: &Manifest, new: &Manifest) -> ManifestDiff`
   - `ManifestDiff` has `added`, `removed`, `changed` for each section
   - `Display` impl for human-readable diff output

6. **crates/manifest/src/adapters/mod.rs** — adapter trait and registry:
   ```rust
   pub mod lang;
   pub mod framework;

   pub trait Adapter: Send + Sync {
       fn name(&self) -> &str;
       fn detect(&self, ctx: &ProjectContext) -> bool;
       fn extract(&self, ctx: &ProjectContext) -> Result<ManifestFragment>;
       fn priority(&self) -> u32;
       fn layer(&self) -> AdapterLayer;
   }

   pub enum AdapterLayer { Language, Framework }

   pub fn all_adapters() -> Vec<Box<dyn Adapter>> { ... }
   ```
   - `all_adapters()` returns all registered adapters (initially empty, adapters add themselves)
   - Sort by priority (higher first), then run matching ones

7. **crates/manifest/src/adapters/lang/mod.rs** — empty, will be populated by adapter beans
8. **crates/manifest/src/adapters/framework/mod.rs** — empty, will be populated

### Merge logic (in lib.rs or a merge.rs)
- `fn merge(fragments: Vec<ManifestFragment>) -> Manifest` — combine all fragments
- Later fragments override earlier ones for same keys (framework > language)
- Vectors are concatenated, maps are merged

### Tests
- Merge: two fragments with overlapping keys merge correctly
- Merge: empty fragment doesn't affect result
- Diff: identical manifests produce empty diff
- Diff: added/removed types detected
- Schema: Manifest serializes to YAML and round-trips
- Context: build from tempdir with some files

### Existing code patterns
The project uses `anyhow` for errors, `clap` for CLI. See `crates/detect/` for how crates are structured — single `lib.rs` with tests at bottom. But this crate is bigger, so split into modules.

### Workspace setup
Add the crate to the workspace in the root `Cargo.toml` (it's already `members = ["crates/*"]` so just creating the dir should work).
