- **`crates/uu/src/cmd/clean.rs:14`** `single-responsibility` — The `execute()` function is 90 lines and handles detection, size calculation, native clean command dispatch, artifact directory removal, and summary output — five distinct responsibilities in one function. Every other command module delegates to `runner::run_steps()` but `clean` reimplements its own execution flow.
  > *Extract helpers: `calculate_artifact_sizes()`, `run_native_clean()`, `remove_artifact_dirs()`. This would make the logic easier to test independently and keep `execute()` as a coordinator.*
- **`crates/uu/src/cmd/clean.rs:135`** `resource-cleanup` — `dir_size` silently swallows all errors by calling `dir_size_inner(path).unwrap_or(0)`. If `read_dir` fails due to permission errors or I/O issues, the function reports 0 bytes instead of warning the user. This means `uu clean` could report "nothing to clean" or undercount freed space when the user lacks read permissions on subdirectories, masking real problems.
  > *Propagate or log errors from `dir_size_inner`. At minimum, emit a warning to stderr when a directory can't be read (e.g., "warning: could not read {path}: permission denied") so the user knows the size estimate is incomplete.*
- **`crates/uu/src/cmd/clean.rs:139`** `edge-cases` — `dir_size_inner` uses unbounded recursion to traverse directory trees. Deeply nested directory structures (e.g., a pathological `node_modules` tree with thousands of nesting levels) could cause a stack overflow and crash the process. This is a denial-of-service vector if `uu clean` is run in a directory with adversarial nesting.
  > *Convert to an iterative approach using an explicit stack (e.g., `Vec<PathBuf>`) or cap recursion depth with a reasonable limit (e.g., 256 levels). Alternatively, use the `walkdir` crate which handles this safely.*
- **`crates/uu/src/cmd/ports.rs:155`** `edge-cases` — `parse_lsof_line` uses `fields.len().wrapping_sub(2)` to index from the end when `(LISTEN)` is the last field. If `fields.len()` is 0 or 1, `wrapping_sub(2)` wraps to `usize::MAX`, and `fields.get()` returns `None` which is safely handled. However, using `wrapping_sub` for defensive arithmetic in a parser is a code smell — it obscures the intent and could mask logic errors in future changes.
  > *Use `fields.len().checked_sub(2)` and explicitly handle the `None` case, or restructure the parsing to use `fields.get(fields.len() - 2)` only when `fields.len() >= 2` is already guaranteed by the earlier length check.*
- **`crates/uu/src/cmd/clean.rs:51`** `graceful-degradation` — When the native clean command (e.g., `cargo clean`) fails, the code logs a warning but continues to remove artifact directories. If `cargo clean` failed because of a locked build directory, the subsequent `remove_dir_all` might also fail with a less clear error. More importantly, if the native clean partially succeeds, the pre-calculated `artifact_sizes` become inaccurate — the "freed" size reported to the user could be double-counted (native clean already freed some, then `remove_dir_all` frees the rest or the dir is already gone).
  > *Recalculate sizes after native clean runs, or skip manual removal for directories that the native clean is expected to handle (e.g., skip `target/` removal when `cargo clean` succeeds). At minimum, note in the output that the freed size is an estimate.*
- **`crates/uu/src/cmd/ports.rs:98`** `error-handling` — `list_listeners` runs `lsof -iTCP -sTCP:LISTEN -nP` but does not handle the case where lsof requires root/sudo to see all listening sockets. On macOS and some Linux configurations, non-root users may not see all listeners. The code warns on non-zero/non-1 exit codes but doesn't inform the user that results may be incomplete without elevated privileges, which could lead to `uu ports --kill 3000` reporting "nothing listening" when a root-owned process is actually using the port.
  > *After a successful lsof invocation, check if the user is non-root and add a hint like "note: some listeners may be hidden — run with sudo for complete results" when the result set is empty or when the user is querying a specific port that shows no results.*
